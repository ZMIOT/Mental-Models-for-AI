# Book Bible

## 项目定位

《Deep Learning Mental Models》不是公式手册，也不是框架教程，而是一本帮助工业 3D 视觉工程师建立深度学习直觉的书。

目标读者已经熟悉工程现场、相机、点云、标定、检测、机器人或自动化系统，但可能缺少对深度学习概念的稳定心智模型。

## 写作原则

1. 先建立直觉，再给数学表达。
2. 先解释为什么，再解释怎么做。
3. 每个抽象概念都要落到工业 3D 视觉场景。
4. 避免为了完整性堆砌知识点。
5. 不把框架 API 当作知识本身。
6. 所有术语以 `GLOSSARY.md` 为准。
7. 所有知识关系以 `KNOWLEDGE_MAP.md` 为准。

## 每章固定模板

具体可复制的章节骨架维护在 `Chapter_Template.md`。本节只规定所有章节必须包含的教学结构。

1. 故事
2. 问题
3. 历史背景
4. 心智模型
5. 图解
6. 数学直觉
7. 最小代码
8. 工业 3D 视觉案例
9. 常见失败案例
10. AI 实验
11. Mini Project
12. 知识地图
13. 思考题
14. 总结

## 语言风格

- 面向工程师，清楚、直接、可落地。
- 可以使用类比，但类比必须服务于概念，不喧宾夺主。
- 数学部分强调直觉，不用公式制造门槛。
- 代码只做最小演示，不追求完整框架工程。
- 工业案例要真实、具体，避免泛泛而谈。

## 图示规范

- 优先使用 Mermaid 表达知识关系、数据流和模型结构。
- 复杂视觉内容可以在 `Figures/` 中放置图片或插图占位。
- 图示必须解释一个明确概念，不能只是装饰。

## 代码规范

- 最小代码优先 Python。
- 代码块必须能独立说明一个概念。
- 不把 PyTorch、TensorFlow 或其他框架写成唯一答案。

## 章节开发前置条件

- 正式章节写作前，必须先确认 `CURRICULUM.md` 中有对应位置。
- 正式章节写作前，必须先创建并确认对应 `ChapterNN_SPEC.md`。
- 章节正文不得突破对应 Spec 中定义的边界。
- 如果正文需要改变边界，先修改 Spec，再修改正文。
