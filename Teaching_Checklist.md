# Teaching Checklist

本文件用于每章进入发布前的教学质量检查。它不是内容模板，而是审校清单。

## 使用时机

每章在以下阶段都应使用本清单：

- Phase 1: Chapter Spec
- Phase 2: Teaching Design
- Phase 3: Draft
- Phase 4: Technical Review
- Phase 5: Teaching Review
- Phase 6: Final

## Phase 1: Chapter Spec

- [ ] 本章在 `CURRICULUM.md` 中有明确位置。
- [ ] 本章有 `ChapterNN_SPEC.md`。
- [ ] 本章说明了讲什么。
- [ ] 本章说明了不讲什么。
- [ ] 本章说明了前置章节和下一章。
- [ ] 本章学习目标可验证。
- [ ] 本章边界不覆盖后续章节的核心职责。

## Phase 2: Teaching Design

- [ ] 本章说明了最想改变读者的哪个错误认知。
- [ ] 本章有一句希望读者一年后仍记住的话。
- [ ] 本章有明确的能力目标，而不仅是知识目标。
- [ ] 本章有一个核心问题。
- [ ] 本章有一个稳定心智模型。
- [ ] 本章有故事入口，而不是从定义开始。
- [ ] 本章有清晰的 Teaching Flow。
- [ ] 本章有实验设计方向。
- [ ] 本章有一个主工业 3D 视觉案例。
- [ ] 本章有常见误解或失败案例。
- [ ] 本章知道哪些概念只埋伏笔，不展开。
- [ ] 本章知道读者读完后应该能做什么。
- [ ] 本章内容没有从 Idea 直接进入 Book Content。

## Phase 3: Draft

- [ ] 正文遵守 `Chapter_Template.md`。
- [ ] 正文遵守 `BOOK_BIBLE.md` 的写作原则。
- [ ] 正文遵守中文优先原则，除专用术语外没有不必要的英文表达。
- [ ] 每个小节服务本章核心问题。
- [ ] 没有把框架 API 当作知识本身。
- [ ] 数学只服务直觉。
- [ ] 工业案例具体、真实、可落地。

## Phase 4: Technical Review

- [ ] 术语与 `GLOSSARY.md` 一致。
- [ ] 知识关系与 `KNOWLEDGE_MAP.md` 一致。
- [ ] 代码示例能解释本章概念。
- [ ] 图示准确表达概念关系。
- [ ] 工业案例没有明显技术错误。
- [ ] 没有引入未解释的关键概念。

## Phase 5: Teaching Review

- [ ] 读者能先建立直觉，再看到数学表达。
- [ ] 类比没有喧宾夺主。
- [ ] 章节开头的问题在结尾得到回答。
- [ ] 本章不会让读者误以为已经学完后续章节内容。
- [ ] 思考题能帮助读者迁移到自己的工程现场。

## Phase 6: Final

- [ ] `STATUS.md` 已更新。
- [ ] 相关术语已更新到 `GLOSSARY.md`。
- [ ] 相关知识关系已更新到 `KNOWLEDGE_MAP.md`。
- [ ] 相关案例已更新到 `CASE_LIBRARY.md`。
- [ ] 章节状态符合 Draft / Review / Revision / Final 流程。
- [ ] `CODEX_REPORT.md` 已记录本轮修改。

## 发布原则

一章不能因为“写完了”就进入 Final。只有当技术准确性、教学清晰度、章节边界和仓库同步都通过检查后，才能进入 Final。
